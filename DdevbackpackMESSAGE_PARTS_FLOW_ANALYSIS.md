# UIMessageStream Message Parts Flow Analysis

## Overview
This document traces how message parts are shaped and passed through the system, from generation to rendering in MCP tool components.

## Data Flow: useChat → ChatMessages → Message Component

### 1. **Initial Message Generation (useChat hook)**
**File**: `src/components/chat.tsx` (lines 146-191)

- Messages have shape: `ChatMessage = UIMessage<MessageMetadata, CustomUIDataTypes, ChatTools>`

### 4. **Message Parts Retrieval & Conversion**
**File**: `src/lib/ai/utils.ts` (lines 32-41)

```typescript
export function convertToUIMessages(messages: Message[]): ChatMessage[] {
  return messages.map((message) => ({
    id: message.id,
    role: message.role as "user" | "assistant" | "system",
    parts: message.parts as UIMessagePart<CustomUIDataTypes, ChatTools>[],
    metadata: {
      createdAt: formatISO(message.createdAt),
    },
  }));
}
```

**Type Cast**: Database JSONB `parts` field is cast to `UIMessagePart[]` here

## Message Part Shapes

### Text Part:
```typescript
{
  type: "text",
  text: string
}
```

### File Part:
```typescript
{
  type: "file",
  mediaType: "image/jpeg" | "image/png",
  filename: string,
  url: string
}
```

### Tool Call/Result/Error Parts (UIMessageStream Format):
```typescript
// tool-call part
{
  type: "tool-call",
  toolCallId: string,
  toolName: string,  // e.g., "mcp_serverName_toolName"
  args?: object
}

// tool-result part
{
  type: "tool-result",
  toolCallId: string,
  result: unknown
}

// tool-error part
{
  type: "tool-error",
  toolCallId: string,
  error?: {
    message: string
  }
}
```

### Built-in Tool Parts (Streaming Format):
```typescript
{
  type: "tool-web_search" | "tool-extract" | etc.,
  toolCallId: string,
  state: "input-available" | "output-available" | "input-streaming",
  input?: object,      // Tool input when state is input-*
  output?: object,     // Tool output when state is output-*
}
```

### MCP Tool Parts (Streaming Format):
```typescript
{
  type: "tool-mcp_serverName_toolName",
  toolCallId: string,
  state: "input-available" | "output-available" | "input-streaming" | "output-error",
  input?: object,      // When available
  output?: object,     // When available
}
```

### Reasoning Part:
```typescript
{
  type: "reasoning",
  text: string
}
```

## Transformations & Filters

### 1. **At Storage (onFinish callback)**
- No transformation - parts stored as-is from AI SDK stream
- Entire message.parts array is persisted to JSONB column

### 2. **At Retrieval (convertToUIMessages)**
- Type casting only: `message.parts as UIMessagePart<CustomUIDataTypes, ChatTools>[]`
- No filtering or transformation of part structure

### 3. **At Rendering (renderMessagePart)**
